# isBounce

Detects whether an email is a bounce (non-delivery report).

## Signature

```ts
import { isBounce } from 'mime-kit';

isBounce(email: Email): boolean
```

## What it does

Checks three signals that indicate a bounce message:

1. **`Content-Type: multipart/report`** — The standard MIME type for delivery status notifications (RFC 3464).
2. **`Return-Path: <>`** — An empty return path means the message was generated by the mail system itself, not a human sender.
3. **`From: mailer-daemon@...`** — The conventional sender address used by MTAs when they generate bounce messages.

Any one of these is enough to return `true`.

## When to use this

**Cleaning up an inbox view.** Bounces aren't real conversations. In a mail client, you might hide them from the main inbox and surface them in a dedicated "Delivery issues" section.

**Email sending infrastructure.** If you run a transactional email service (password resets, invoices, notifications), you need to process bounces to:

- Remove invalid addresses from your sending list
- Track your bounce rate (ISPs will throttle you above ~2%)
- Distinguish hard bounces (address doesn't exist) from soft bounces (mailbox full)

**Preventing auto-reply loops.** Before sending an auto-reply or out-of-office response, check `isBounce()`. You must never auto-reply to a bounce — it creates a mail loop that can generate thousands of messages.

## Examples

### Filtering bounces in an inbox

```ts
import { parse, isBounce } from 'mime-kit';

const emails = rawMimeMessages.map(parse);

const inbox = emails.filter((e) => !isBounce(e));
const bounces = emails.filter((e) => isBounce(e));

console.log(`${bounces.length} delivery failures`);
```

### Bounce processing for a sending service

```ts
import { parse, isBounce } from 'mime-kit';

function handleIncoming(rawMime: string) {
  const email = parse(rawMime);

  if (isBounce(email)) {
    // Extract the original recipient from the bounce body
    // and mark them as undeliverable in your database
    const failedAddress = extractBouncedAddress(email);
    markAddressAsBounced(failedAddress);
    return;
  }

  // Normal email processing...
}
```

### Safe auto-reply guard

```ts
import { parse, isBounce, isAutoReply } from 'mime-kit';

function shouldAutoReply(rawMime: string): boolean {
  const email = parse(rawMime);

  // Never auto-reply to bounces or other auto-replies
  if (isBounce(email)) return false;
  if (isAutoReply(email)) return false;

  return true;
}
```

This guard is critical. Without it, an out-of-office message replying to a bounce triggers a new bounce, which triggers another out-of-office, and so on.
